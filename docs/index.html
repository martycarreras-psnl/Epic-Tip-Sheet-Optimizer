The Problem

Epic tip sheets are invisible to AI

Training documents are packed with annotated screenshots that carry critical contextual information. AI agents can't see them — and when they try to answer without that context, they get it wrong.

🔍

Unsearchable

Agents can't reason over images embedded in PDFs or Word documents. When the answer lives in a red-boxed screenshot annotation, it's completely invisible.

⚠️

Inaccurate

Agents that attempt to answer from partial text hallucinate the missing visual context — confidently giving wrong answers about buttons, fields, and workflows they can't actually see.

🔄

Unmaintainable

When Epic releases updates, there's no structured way to determine which of the thousands of tip sheets are impacted and need revision.

How It Works

End-to-end, from PDF to Copilot agent

A two-track process powered by M365 Copilot Cowork. Convert tip sheets, validate with evals, and ground your Copilot agents — all in one flow.

1

Select Tip Sheet

Choose an Epic tip sheet from the hundreds maintained by your training team — Word or PDF source.

2

Convert with Cowork

M365 Copilot Cowork converts the tip sheet (content + screenshots) into a PDF optimized for LLM consumption.

3

Ingest & Structure

The Tip Sheet Ingestion skill extracts structured steps, OCRs screenshots, captures UI context, and normalizes everything into a consistent schema.

4

Generate Evals

Automatically generate a 20-question eval set across 5 categories with expected answers and scoring rubrics.

5

Run & Validate

Execute evals against the content. Each prompt is tested blind, then graded — producing a scorecard with pass/fail and evidence.

6

Ground Your Agent

Publish validated content to SharePoint. Your M365 Copilot agent is grounded in this library and delivers trusted, accurate answers.

Copilot Skills

Purpose-built skills that do the heavy lifting

Two skills work together: one converts documents with zero-hallucination discipline, the other validates quality with rigorous evals.

INGESTION

Training Content to Knowledge Base

Converts training tip sheets into structured, LLM-readable knowledge base content with strict zero-hallucination extraction.

  • Page-by-page verbatim extraction — no paraphrasing
  • Screenshot → textual description with annotation boundaries
  • ENUM generation for constrained value sets (buttons, tabs, dropdowns)
  • Mandatory verification pass against source
  • Eval question generation (20 questions, 5 categories)
VALIDATION

Eval Runner

Runs structured evals against content to measure how well an LLM can reason over it — with anti-leakage safeguards.

  • Two-stage evaluation: blind answer → informed grade
  • Per-question scoring with rubric matching (0–3 scale)
  • Hallucination detection and flagging
  • Category breakdowns across 5 dimensions
  • Actionable recommendations when content fails

2

3

Identify Weak Areas

4

Improve the Content

Fix the flagged sections — enrich descriptions, add missing ENUMs

5

Repeat until ≥80% pass threshold. Then publish to SharePoint.
